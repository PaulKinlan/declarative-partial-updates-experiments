# 08 · skeleton card

A product card whose placeholder shapes match the final layout exactly — no layout shift when the
data arrives.

The point: a generic "loading..." or three identical bars is OK for prototypes, but in production
you want the skeleton to act as a visual table of contents for what's about to fill in. Same image
dimensions, same title width, same button height. The eye sees the page is already done; only the
content is missing.

Adapted from
[patching-user-data](https://github.com/PaulKinlan/3d-io-demo-26/tree/main/demos/patching-user-data)
in `3d-io-demo-26`. Same idea, different runtime, shared CSS variables.
