09. replace, don't diff (do we still need the virtual DOM?)

A live leaderboard that reshuffles every second. The server re-renders the whole sorted list and streams it as a <template for="board"> that replaces the region in place. No virtual DOM, no diffing, no reconciliation, no framework, and no client JS. Each row carries a stable view-transition-name, so a browser that opts into view transitions animates the reshuffle for free.

  1. 1 Astro Foxes 42
  2. 2 Quantum Owls 39
  3. 3 Neon Yaks 37
  4. 4 Pixel Pandas 35
  5. 5 Turbo Newts 31
  6. 6 Velvet Crabs 28

Open DevTools' Network panel: it's one response that never finishes, flushing a fresh <template for="board"> each tick. Without the API you'll see this first board as a static snapshot instead of a live one.

The virtual DOM exists to avoid re-rendering everything: diff the new render against the old tree, then patch only what changed. When the platform can replace a whole named region from streamed HTML, a large class of "you need a framework for this" UIs become a server that prints HTML.
